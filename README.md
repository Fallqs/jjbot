# JJBot - 晋江文学城镜像阅读器

纯净阅读体验的晋江文学城镜像站，支持免费章节和VIP章节阅读。

## 功能特性

- 📚 小说目录浏览与搜索
- 📖 沉浸式阅读体验
- 🎨 多种阅读主题（白天/夜间/护眼/清新）
- 🔤 字体大小与行间距调节
- 🔐 VIP章节支持（需登录晋江账号）
- 🍪 Cookie自动解析（支持多种格式导入）
- ⚡ 章节预加载与缓存（自动缓存前后章节，5分钟TTL）

## 技术栈

- **前端**: React + TypeScript + Vite + Tailwind CSS
- **后端**: Express + Puppeteer（用于VIP章节渲染）
- **UI组件**: shadcn/ui

## VIP章节阅读说明

VIP章节需要晋江文学城账号Cookie才能阅读。获取方式：

1. 在浏览器中登录 [jjwxc.net](https://www.jjwxc.net)
2. 购买需要阅读的VIP章节
3. 按 F12 → Application → Cookies → `jjwxc.net`
4. 复制Cookie字符串并粘贴到应用设置中

## VIP章节渲染流程

VIP章节使用自定义字体进行文本混淆。本应用通过以下流程实现正确渲染：

```
┌─────────────────────────────────────────────────────────────────┐
│                         渲染流程                                 │
└─────────────────────────────────────────────────────────────────┘

  浏览器请求VIP章节
         │
         ▼
  ┌──────────────┐     ┌──────────────────┐     ┌─────────────────┐
  │  Vite Dev    │────▶│ Puppeteer Server │────▶│  JJWXC VIP Page │
  │  (Port 5173) │     │   (Port 3001)    │     │ (my.jjwxc.net)  │
  └──────────────┘     └──────────────────┘     └─────────────────┘
         │                      │
         │                      ▼
         │              1. 使用Cookie登录
         │              2. 等待页面解密
         │              3. 提取HTML + CSS
         │              4. 修改字体URL为代理地址
         │                      │
         │◀─────────────────────┘
         │
         ▼
  返回包含PUA字符的HTML和@font-face CSS
         │
         ▼
  浏览器请求字体文件
         │
         ▼
  ┌──────────────┐     ┌──────────────────┐     ┌─────────────────┐
  │  Vite Proxy  │────▶│  /proxy-font/    │────▶│  static.jjwxc   │
  │  /proxy-font │     │   Endpoint       │     │   .net          │
  └──────────────┘     └──────────────────┘     └─────────────────┘
         │                      │
         │◀─────────────────────┘
         │
         ▼
  字体加载成功！
  PUA字符 (如 \uE832) 通过字体映射显示为正确汉字 (如 "人")

┌─────────────────────────────────────────────────────────────────┐
│                      关键技术点                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. PUA字符 (Private Use Area)                                   │
│     - JJWXC使用U+E000-U+F8FF范围的Unicode字符替代真实汉字        │
│     - 例如: \uE832 在视觉上显示为 "人"                           │
│                                                                  │
│  2. 自定义字体 (@font-face)                                      │
│     - 通过WOFF2字体文件定义PUA字符的字形                         │
│     - 字体文件中的cmap表: U+E832 → 字形ID #1234                  │
│     - 字形 #1234 的轮廓是 "人" 的形状                           │
│                                                                  │
│  3. 渲染 vs 文本提取                                             │
│     - 浏览器渲染: 显示正确的汉字形状                             │
│     - DOM文本提取: 仍然得到PUA字符 \uE832                        │
│     - 这就是为什么我们需要完整的渲染环境                         │
│                                                                  │
│  4. 字体代理                                                     │
│     - 由于CORS限制，直接从JJWXC加载字体会被阻止                  │
│     - 后端代理字体文件，前端从同源地址加载                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 章节缓存与预加载

为提升VIP章节的阅读体验，应用实现了智能缓存系统：

### 缓存策略

- **TTL**: 30分钟自动过期
- **容量**: 最多缓存10章（LRU淘汰）
- **范围**: 自动缓存当前章节的前后各1章

### 工作流程

```
用户阅读第N章
       │
       ▼
┌──────────────┐
│ 加载当前章节  │
│ (优先从缓存)  │
└──────────────┘
       │
       ▼
┌──────────────┐     ┌──────────────┐
│ 后台预加载    │────▶│  第N-1章     │
│              │     │  (上一章)    │
│              │────▶│  第N+1章     │
│              │     │  (下一章)    │
└──────────────┘     └──────────────┘
```

### 缓存指示器

- 导航按钮上的**绿色圆点**表示该相邻章节已缓存
- 章节标题下方显示**已缓存章节数量**

### 技术实现

```typescript
// Reader.tsx
const chapterCache = new Map<number, CachedChapter>();
const CACHE_DURATION = 30 * 60 * 1000; // 30分钟
const MAX_CACHE_SIZE = 10;

// 自动预加载相邻章节
const prefetchAdjacentChapters = async (currentId: number) => {
  const idsToFetch = [currentId - 1, currentId + 1].filter(...);
  await Promise.all(idsToFetch.map(id => fetchAndCacheChapter(id)));
};
```
